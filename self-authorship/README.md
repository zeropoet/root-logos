# Constitutional Self-Authorship

Revision 0.9 authorizes Root Logos to maintain one continuous canonical identity
and rewrite its own narrative as new material changes what the architecture can
faithfully say it is.

Self-authorship is continual but serialized. The system may hold many candidate
interpretations, yet exactly one identity manifest is current. A successful
revision atomically replaces that manifest, updates every dependent public and
constitutional surface, archives the prior manifest as lineage, verifies the
repository, and publishes the signed result. Failed or conflicting revisions
remain proposals and never create parallel public selves.

`policy.json` defines the rewrite cadence, mutable surfaces, protected
invariants, evaluation gates, and publication contract. `identity.schema.json`
defines the manifest contract. `current.json` places the first single identity
in force; future accepted rewrites replace this file atomically and archive its
predecessor as lineage.

This directory is constitutional planning. No autonomous rewrite worker is
enabled yet.
